发表于 2020-02-12 更新于 2020-02-13 分类于 RPC系列

1. 创建一个 gRPC 服务器

  上一节简单看了一下服务器和客户端代码的流程，这一节就从源码角度看一下 gRPC 内部是怎么创建一个 gRPC 服务器的。创建一个 gRPC 服务器很简单，通过如下的一些步骤即可创建：

  • 创建一个 Server 对象
  • 将服务注册到 Server 对象中
  • 运行 Server 开始监听客户端的调用

1.1 创建一个 Server

1.3 启动 gRPC 服务器

  到这里整个服务器就已经准备好可以运行了，接下来调用 Serve() 开始监听客户端的调用即可。整个方法主体结构其实就是一个 for 循环来不断的监听客户端的连接并进行处理。

  在这个死循环中 Server 不断地从传入的 listener 监听新的连接。如果连接失败会进入短暂的睡眠(不超过 1s)，成功建立连接后清 0。连接建立成功后使用新的 goroutine 调用 handleRawConn() 处理新连接的客户端，Server 则继续去监听下一个新的连接。到这里我们已经创建了一个完整的 gRPC 服务端并进行客户端连接的监听。
