  gRPC 使用的是经典的 Client/Server 模式，整个基本的使用过程简单来说分为三步：

  1. 编写 .proto 文件来定义服务
  2. 使用 protocol 工具生成客户端和服务端使用的代码
  3. 使用 gRPC 的 API 编写客户端和服务端的逻辑代码

简单接口的调用

  一行代码就实现了简单 rpc 的调用，通过客户端直接执行 GetFeature 方法即可。这就是 RPC 框架的目标，调用远程服务就像本地服务一样简单。

服务器流式接口的调用

  整个代码逻辑也很简单，同样是使用 client 直接调用接口但是返回的却是一个 RouteGuide_ListFeaturesClient 数据流对象，然后在 for 循环中不断的调用 stream.Recv() 获取数据并判断是否收到 EOF 即可。

客户端流式接口的调用

  和前面不同的是，客户端流式接口的调用会在执行完 client.RecordRoute 后返回一个 stream 对象，然后客户端不断地通过 stream.Send() 将数据传送给服务器，在最后通过 stream.CloseAndRecv() 接收服务器返回并关闭数据流。

双向流式接口的调用

  相比于前面的接口调用，双向流传输明显要复杂一些。执行 client.RouteChat 返回一个数据流对象，客户端和服务器均通过这一个对象进行数据的传输，但是这里使用了一个 goroutine 来异步的发送和接收数据，二者之间并不影响。发送和接收分别就是 Send() 和 Recv() 函数。最后客户端主动调用 CloseSend() 关闭写操作，并等待从 stream 中读取 EOF 才结束执行。
